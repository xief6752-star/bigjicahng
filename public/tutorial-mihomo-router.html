路由器 2025-09-25

OpenWrt + Mihomo
旁路由透明代理教程

旁路由方案可以让家里所有设备无需单独配置，直接走代理。适合有折腾能力的用户，配置成本高但用起来最省心。本文介绍 OpenWrt 安装 Mihomo（原 Clash Meta）的完整流程。
⚠️ 高级教程，有一定门槛
本教程需要你有一台运行 OpenWrt 的路由器或软路由，具备基础 Linux 命令行操作能力。配置出错可能导致断网。建议有经验的用户操作，新手请先用手机/电脑客户端方案。

方案说明

旁路由方案：主路由不动，用一台运行 OpenWrt 的设备（软路由/树莓派/小主机）作为旁路网关，家中设备将网关地址指向旁路由，流量经过 Mihomo 分流后再出去。

推荐硬件：R2S / R4S / N100 小主机 / 旧电脑安装 OpenWrt，或直接购买预装 OpenWrt 的软路由设备。

安装 Mihomo

SSH 登录 OpenWrt
在终端执行 ssh root@192.168.1.1（IP 根据实际修改），输入密码登录。
安装 luci-app-mihomo
推荐使用 luci-app-mihomo 插件，提供图形化界面：
opkg update opkg install luci-app-mihomo

如果 opkg 中没有，可手动下载 ipk 包安装，或使用第三方软件源（如 kiddin9 源）。

配置 Mihomo

上传订阅配置
在 OpenWrt 后台 → 服务 → Mihomo → 「配置文件」→ 输入机场提供的 Clash Meta 订阅链接 → 下载配置。
配置透明代理
在 Mihomo 设置页中开启「透明代理」，设置旁路由 IP 作为网关地址，选择 TProxy 模式（推荐）。
修改客户端网关
家中设备的网络设置 → 将「默认网关」改为旁路由的 IP 地址 → 即可走透明代理，无需在设备上安装任何软件。
DNS 防泄漏：同时将客户端 DNS 也改为旁路由 IP，让 Mihomo 接管 DNS 解析，避免 DNS 泄漏导致分流失效。

验证是否生效

在家中任意设备浏览器访问 ip.sb，如显示代理节点的 IP 地址，说明透明代理配置成功。
